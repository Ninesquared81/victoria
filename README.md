# victoria
A C-like systems programming language for my own needs

## Project overview

This is the reference compiler project for the Victoria programming language.

The project includes two compilers: a boostrap compiler, written in C, which compiles only a subset of
Victoria, known as rVic; and the full Victoria compiler, written in rVic. The main use case of the bootstrap
compiler is to compile the main compiler.

### Getting the compiler

There are currently no pre-built binaries, so the only way to get the compiler is to build from source and
bootstrap the compiler.

### Bootstrapping

> [!NOTE]
> The main compiler is currently unfinished. Use the bootstrap compiler to compile rVic code.

To bootstrap the compiler, you can run `make bootstrap` in the project root directory (here), or simply
run `make` in the ./bootstrap/ directoy. This automates the boostrappping process.

#### Manual bootstrapping

Alternatively, you can manually bootrap the compiler. Simply compile all the .c sources in ./bootstrap/
into a single executable (called crvic/crvic.exe by the Makefile, but you can give it any name).

For instance:

```shell
cc *.c -o crvic
```

Then, use that program to build the source code in ./src/ and then compile the output with you C compiler:

```shell
./bootsrap/crvic ./src/main.vic
cc ./src/main.c ./bootstrap/lexer.c -o ./bin/vic
```

#### Testing the bootstrapped compiler

> [!WARNING]
> This will not work at the moment because the main commpiler is unfinished.

> [!NOTE]
> The c code generated by the compiler depends on ./bootstrap/lexer.c, so isn't fully self-hosted. This is
> because lexing is currently handled by my lexing library [lexel](https://github.com/Ninesquared81/lexel),
> and lexer.c is a convenient wrapper for calling this from Victoria.

You can test that the bootstrapping process worked by using the main compiler to compile itself:

```shell
./bin/vic ./src/main.vic
cc ./src/main.c ./bootstrap/lexer.c -o ./bin/vic1
mv ./bin/vic1 ./bin/vic
```

# Language Ideology

This is a brief summary of the "big ideas" of Victoria.

- The language should not be too ideological.

- Trust the programmer.

- Statements are statements and expressions are expressions; statements and expressions should be
clearly distinguished.

- Obvious things should be simple; complicated things should be clear.
